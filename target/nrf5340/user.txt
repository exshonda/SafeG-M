    TOPPERS/SafeG-M
    nRF5340依存部 ユーザマニュアル

    対応バージョン: Release 1.0.1
    最終更新: 2022年04月29日

○概要
本ターゲット依存部でサポートするターゲットプロセッサ，評価ボード，ターゲ
ット名は次の通りである．

・Nordic nRF5340
 ・評価ボード   : nRF5340-DK
 ・ターゲット名 : nrf5340_dk_gcc

本マニュアルでは上記ターゲット上でSafeG-Mのサンプルプログラムを動作させる
方法を説明する．ターゲットに関するその他の情報はasp3/target/nrf5340_dk_gcc/
target_user.txtに記載されているため，こちらも参照すること．

○Zephyrのセットアップ
本ターゲットではゲストOSとしてZephyrの実行をサポートしている．動作を
確認しているZephyrのバージョンは以下の通り．

・Zephyr v2.6.0-rc1-ncs1

なお，Zephyr単体ではなくnRF Connect SDKを利用することを想定している．
動作を確認したConnect SDKのバージョンは以下の通り．

・Nordic Connect SDK v1.6.1

nRF Connect SDKとZephyrのセットアップについては以下のドキュメントが
参考になる．ただし，GNのインストールは必須ではない．
https://developer.nordicsemi.com/nRF_Connect_SDK/doc/latest/nrf/gs_installing.html

例えば，以下のコマンドにより上記バージョンのZephyr及びConnect SDKを
入手できる．インストールするディレクトリに特に制限はない．

$ west init -m https://github.com/nrfconnect/sdk-nrf --mr v1.6.1
$ west update

○サンプルプログラムのビルド方法

1.
secureディレクトリでmakeを実行し，asp3のサンプルをビルドする．
また，make runを実行してターゲットに書き込む．

2.
Zephyrの環境を設定する．典型的にはZephyrのインストールディレクトリに
移動し，以下のようなコマンドを実行する．

$ export ZEPHYR_TOOLCHAIN_VARIANT=gnuarmemb
$ export GNUARMEMB_TOOLCHAIN_PATH=<path/to/toolchain>
$ export BOARD=nrf5340dk_nrf5340
$ source zephyr-env.sh

これらZephyrの環境変数については公式のリファレンスも参照されたい．
なお，2022年6月現在，WSLはサポートされていないようである．
https://docs.zephyrproject.org/latest/develop/toolchains/gnu_arm_embedded.html

3.
non-secureディレクトリで以下を実行し，Zephyrのサンプルをビルドする．

$ west build -b nrf5340dk_nrf5340_cpuappns -- -DCONFIG_SPM=n -DCONFIG_FPU=y

4.
以下のコマンドを実行し，Zephyrサンプルを書き込む．

$ west flash
