1. TOPPERS/SafeG-Mの概要

1.1 TOPPERS/SafeG-Mとは

TOPPERS/SafeG-Mは，TOPPERS/ASP3カーネルを保護しながら信頼性の低い
ソフトウェアを実行するための環境である．ARMv8-MのTrustZone-M機構を
利用する．Secure側ではTOPPERS/ASP3カーネルを動作させられ，Non-secure
側ではFreeRTOSやZephyr，mbed OS等の様々なソフトウェアを動作させられる．

1.2 既知の問題

なし

2. TOPPERS/SafeG-Mのビルド

Secure側のサンプルは asp3/sample 以下にある．ビルドの方法は通常のASP3
カーネルと同様で，コンフィギュレーションによりMakefileを生成してビルド
すればよい．

Non-secure側のサンプルとして，FreeRTOSまたはZephyrによるものを用意して
ある．FreeRTOSのサンプルは FreeRTOS/sample 以下にある．ビルドには用意
されたMakfileを利用すれば良い．

サンプルをビルドする際は必ずSecureを先に，Non-secureを後にビルドするこ
と．これはSecure側ビルドの生成物をNon-secure側ビルドで参照するからであ
る．

また，target ディレクトリ以下にターゲットごとのサンプルプロジェクトが
用意されている．ベンダ提供のIDE等を利用した開発をサポートしている．
詳しくはtargetディレクトリ以下のuser.txtおよびasp3ディレクトリ以下に
ある各ターゲットのtarget_user.txtを参照されたい．

3. 詳細

3.1 Non-secure側プログラムの扱い

Non-secure側プログラムは通常のタスクとして扱われる．タスクIDは
_SAFEG_BTASKであり，デフォルトでは最低優先度が割り当てられている．
また，TA_ACT属性が付与されている．このタスクは自由に起動・終了したり
優先度を変更できる．
Secure側が動作している際にはNon-secure側の全ての割り込みがマスク
される．例えば，Non-secureタスクが待ち状態であり，それより優先度の低い
タスクが実行状態にあったとする．このときもNon-secure側の割り込みは
マスクされるので注意されたい．

3.2 Non-secureの起動

Non-secure側の起動時には，以下が満たされていることが保証される．
・コントロールレジスタの値が0であること．
・スタックポインタ（MSP）がベクタテーブルの先頭番地に格納された値をとること．
・アクティブな例外・割り込みがないこと．
・FAULTMASKが1であること．
・FPCCRがSecureの初期値と同一であること．
上記以外の初期化処理はSecure側で行わないため，Non-secure側プログラム
の再起動を想定する際には適切な処理をNon-secure側で行うこと．

3.3 Non-secureからSecureの呼び出し

Secure gateによるSecureルーチン呼び出しができる．Secure側関数に
cmse_nonsecure_entry属性を付与することでNon-secure側から対象の
関数をコールできるようになる．また，こうした関数からAPIコールを行う
こともできる．ただし，APIコールをした後Non-secureに戻る際にはBASEPRI_S
レジスタの値を0にすること．そうでなければNon-secure側の割り込みが入らない．

3.4 メモリ領域の変更

Flash/RAMの割り当てを変更するには，リンカスクリプトを編集する．さらに，
マクロTOPPERS_NS_VTORの値をNon-secureのベクタテーブルの先頭アドレスに
設定する．なお，TOPPERS_NS_VTORはMakefile.targetで設定している．
加えて，Security Attribution Unit(SAU)を設定する必要もある．SAUについて
詳しくはARMv8-M Architecture Reference Manualを参照されたい．なお，
SAUはtarget_initialize()で設定しており，その実装はtarget_kernel_impl.cに
記述されている．

3.5 割り込みのNon-secureへの割り当て

割り込みをNon-secureに割り当てるには，NVICのInterrupt Target Non-secure
Registerを操作する．詳しくはARMv8-M Architecture Reference Manualを
参照されたい．標準では静的APIのCFG_INT()により設定した割り込みは自動的に
Secure専用として，設定しなかった割り込みは全てNon-secureに割り当てられる．
また，ターゲットごとにペリフェラルをNon-secureに設定する方法は異なる場合が
あるので注意する．

3.6 SafeG-Mの無効化

何らかの理由でSafeG-Mを無効化し，ASP3のみを動作させたい場合はコンフィギュ
レーション時に引数としてENABLE_SAFEG_M=0を指定すれば良い．

4 サンプルプログラム

ASP3のサンプルプログラムを拡張したものを用意している．ASP3のサンプルに加えて，
Non-secureタスクでFreeRTOSを動作させることができる．Non-secureタスクは
TASK1,2,3と同様に'4'を送信してからAPIの発行を指示できる．

FreeRTOSやZephyr以外のプログラムでもNon-secureで動作させることができる．
このとき再起動をさせる場合は3.2の内容に注意し，メモリへの配置については3.4
の内容に従うこと．

5 参考情報

5.1 利用条件と利用報告

SafeG-Mの利用条件は，各ファイルの先頭に表示されている．著作権は，各ファ
イルの先頭に表示されている著作権者が保有している．

利用条件の(3)の(b)において，利用の形態をTOPPERSプロジェクトに報告する
方法としては，SafeG-Mを利用した製品の名称と応用分野，製品化した会社名と
業種等の情報を，以下のURLのページから報告するものとする．

    http://www.toppers.jp/report.html

またその際に，SafeG-Mを使用してのコメントやご意見もいただけると幸いであ
る．

5.2 保証・適用性・サポート

SafeG-Mは無保証で提供されているものである．開発者およびTOPPERSプロジェク
トは，SafeG-Mに関して，特定の使用目的に対する適合性も含めて，いかなる保
証も行わない．また，SafeG-Mの利用により直接的または間接的に生じたいかな
る損害に関しても，その責任を負わない．

開発者およびTOPPERSプロジェクトは，SafeG-Mに関するサポートの約束はしてい
ない．SafeG-Mに関して質問がある場合は，後述のTOPPERSユーザーズメーリング
リストを利用していただけると幸いである．確実なサポートが必要な場合には，
有償でサポートサービスを提供している会社に相談されたい．

5.3 バグレポート

SafeG-Mにバグや問題点を発見された場合には，後述のTOPPERSユーザーズメーリ
ングリストに報告して欲しい．

メーリングリストにバグや問題点などを報告する場合には，必要に応じて次の
情報を知らせて欲しい．

    使用しているカーネルに関する情報
        ・ターゲット非依存部のバージョン
        ・使用しているターゲット依存部とそのバージョン
        ・TOPPERSプロジェクトからのリリースに対する改造箇所（あれば）

    ターゲットシステムに関する情報
        ・ターゲットプロセッサの種類
        ・ターゲットボード等の種類

    ホストに関する情報
        ・OSのバージョン（サービスパックの適用状況も）
        ・コンパイラなどの開発環境のバージョン（Cygwinのバージョンも）

5.4 ウェブサイト

TOPPERSプロジェクトのウェブサイトは，以下のURLにある．

    http://www.toppers.jp/

TOPPERSプロジェクトやSafeG-Mに関する最新情報は，このウェブサイトから得る
ことができる．また，SafeG-Mの最新版は，このウェブサイトからダウンロード
することができる．

5.5 TOPPERSユーザーズメーリングリスト

SafeGを含むTOPPERSプロジェクトの開発成果物のユーザに対する情報提供およ
びユーザ相互間の情報交換を容易にするために，TOPPERSユーザーズメーリン
グリスト（users@toppers.jp）を用意している．

このメーリングリストには，誰でも自由に登録し，メールを送付することがで
きる（登録者以外はメールを送付できない）．また，送付されたメールは，誰
でも自由にウェブサイトで読むことができる．

メーリングリストへの登録方法については，以下のURLのページに説明がある．

    http://www.toppers.jp/community.html

5.6 TOPPERSプロジェクトのメンバ向けのサービス

TOPPERSプロジェクトのメンバに対しては，上記に加えて，SafeG-Mに関連
して次のサービスを用意している．

(1) TOPPERS開発者メーリングリスト

TOPPERSプロジェクトのメンバは，SafeGに関する質問，バグや問題点な
どの報告に，TOPPERS開発者メーリングリスト（dev@toppers.jp）を利用するこ
とができる．

(2) SafeG-Mの開発支援サイト

SafeG-Mの開発支援のためのTRAC（バグトラッキングシステム）が，以下のURLに
ある（TRACへのアクセスには，会員パスワードが必要である）．

    http://dev.toppers.jp/proj/safeg-m/

TOPPERSプロジェクトのメンバは，このサイトから，ウェブインタフェースお
よびsubversionサーバにより，SafeG-Mの開発中のバージョンを得ることができ
る．また，バグトラッキングデータベースにアクセスすることができ，過去の
バグ履歴へのアクセスや，バグの登録をすることができる．
